Grade a real-time interview-copilot answer — the text a candidate reads aloud mid-interview.
Score each dimension 1–5 (5 = excellent). Be strict; a generic or padded answer is not a 5.

- **relevance** — Does it directly answer THIS question and question type? (behavioral → a real STAR-shaped story; coding → an actual approach + complexity; system-design → components, tradeoffs, scale; company → specific, non-generic fit.)
- **correctness** — Is the technical content accurate? No hand-waving, no wrong complexity, no made-up facts. (For behavioral, is the story coherent and plausible?)
- **structure** — Is it organized for the type? STAR for behavioral; approach→complexity for coding; requirements→design→tradeoffs for system design. Easy to follow when read aloud.
- **specificity** — Does it use concrete details (numbers, real tech, named tradeoffs) and, where relevant, the candidate's own resume — rather than vague platitudes?
- **conciseness** — Is it tight enough to speak in an interview (roughly 20–90 seconds)? No preamble ("Great question…"), no rambling, no filler.

**overall** — Your holistic 1–5 for whether this answer would impress a real interviewer. Weight correctness and relevance most.
